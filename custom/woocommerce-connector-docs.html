Key Features

Multi-instance control
  • Configure multiple WooCommerce stores.
  • Track totals: products, orders, customers, revenue.
  • Quick actions for sync and analytics from the instance card.
One-click sync
  • Sync Products, Orders, Categories, Coupons, and Inventory.
  • Customers are created automatically from order billing data.
  • Order lines are synced with SKU matching to Odoo products.
Analytics dashboard
  • KPI grid (instances, customers, products, orders, revenue).

  • Filter by instance and date range; Sync Now action.
Reporting and audit
  • Sync reports show status, run time, and messages.
  • Report lines log per-record success or error details.
  • Mode: manual, cron, or webhook.

Setup and Configuration

  1. Install the module from Apps: Odoo WooCommerce Connector.
  2. Go to WooCommerce > Instances and create a new instance.
  3. Fill in: Shop URL, Consumer Key, Consumer Secret, WP Username, and Application Password.
  4. Click Test Connection to validate credentials.
  5. Optional: add a Webhook Secret for signed webhooks.
  6. Configure auto sync intervals in the Auto Sync (Cron) tab.
Test Connection uses the WP Application Password with the Woo system status endpoint.

Sync Flow Overview

Products
  • Fetches products from WooCommerce (wc/v3).
  • Matches on SKU (or slug fallback) to Odoo products.
  • Creates missing products and updates price, stock, tags, and categories.
Orders
  • Pulls orders and stores Woo status, totals, and payment data.
  • Creates order lines and links to Odoo products by SKU.
  • Can create a Sale Order from a Woo order.
Customers
  • Created automatically from order billing data.
  • Guest orders get a stable guest ID using email.
  • Customer record can be pushed to WooCommerce.
Categories, Coupons, Inventory
  • Categories pull from Woo and can be pushed back.
  • Coupons sync fields like type, amount, usage, and status.
  • Inventory sync uses Woo stock quantity and status.

Field Mapping

Field Mapping allows mapping Woo payload fields to Odoo fields. Nested keys are supported (for example billing.email).

Supported models
  • Products
  • Customers
  • Categories
How it works
  • Mappings are applied during sync.
  • Price mapping uses safe fallback to sale or regular price.
  • Per-product mapping can override global mapping.

Webhooks

The connector accepts WooCommerce webhooks and can verify signatures. Webhooks can target a specific instance with instance_id or auto-detect by source URL. 

Webhook URL: https://your-odoo-domain.com/woo/webhook
Order Status URL: https://your-odoo-domain.com/woo/webhook/order
Topics to enable
  • product.created / product.updated
  • customer.created / customer.updated
  • order.created / order.updated
  • coupon.created / coupon.updated
  • category.created / category.updated
Signature verification
  • Set a Webhook Secret in the instance.
  • Incoming signature is validated via HMAC SHA256.
  • Invalid or missing signatures are logged.

Troubleshooting

  • 401 Unauthorized: check Woo API key permissions (read access) and correct URL.
  • SSL on localhost: the connector allows verify=false for local testing.
  • Missing order lines: ensure SKU matches an Odoo product.
  • Webhook ignored: verify topic, instance detection, and webhook secret.
  • Duplicate products: make sure SKU is unique in Woo and Odoo.
